## 📚 Idea/Concepto 
Las **Historias de Usuario** son descripciones breves y sencillas de una funcionalidad, redactadas desde el punto de vista del usuario final. No son documentos técnicos largos; más bien son una forma "ligera" de capturar requisitos justo cuando se necesitan, usando a menudo un formato simple: "Como [quién], quiero [qué], para [para qué]". 
Lo más importante de una historia de usuario no es el texto escrito en la tarjeta, sino las conversaciones constantes que genera entre el equipo de desarrollo y el Product Owner para aclarar los detalles
