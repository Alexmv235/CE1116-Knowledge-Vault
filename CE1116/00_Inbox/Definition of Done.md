## 📚 Idea/Concepto 

La **Definición de Terminado (DoD)** es como una lista de verificación (checklist) que el equipo debe completar totalmente antes de decir que una tarea o funcionalidad está realmente lista para entregarse al cliente. No basta con solo escribir el código; para estar terminado. El trabajo debe haber pasado por todas las etapas necesarias como diseño, construcción, integración, documentación y, muy importante, pruebas. Tener un DoD claro evita sorpresas desagradables al final del proyecto, asegurando que cada incremento del producto tenga una calidad real y validada desde el principio.
