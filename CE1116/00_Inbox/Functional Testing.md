## 📚 Idea/Concepto 

Las **Pruebas Funcionales** se encargan de verificar que el sistema haga exactamente lo que los requerimientos funcionales o del usuario indican. Se consideran pruebas de caja negra, ya que al evaluador solo le interesa si el sistema responde correctamente a ciertas entradas (inputs) con las salidas esperadas (outputs), sin importar cómo esté programado el código internamente. Diseñar estas pruebas desde el inicio es fundamental porque ayuda a aterrizar la visión del producto y permite detectar errores, vacíos o ambigüedades en los requerimientos mucho antes de que el software esté terminado
## 📌 Puntos Claves (Opcional)
- Deben ser transparentes, solo importa que la entrada y la salida correspondan con el comportamiento esperado.
